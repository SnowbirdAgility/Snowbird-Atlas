02 Layout & Visual Hierarchy

Section-by-section.

  • Rhythm — alternate light (Cloud White) and dark (Midnight Navy) bands so the page breathes: Hero (dark) → Capabilities (light) → Why Us (dark) → Approach (light) → Outcomes (white, ruled) → Credibility (dark) → Final CTA (gradient card on light) → Footer (deep navy).
  • Hero — two columns on desktop (≈58% copy / 42% visual). Headline is the largest type on the page; one cyan-highlighted word (“execution”) is the only accent. Two CTAs: solid cyan primary, outlined secondary.
  • Capabilities — 3×2 card grid. Each card: small geometric icon → mono index → bold title → 2-line description. Hover lifts the card and brings in a cyan border.
  • Why Us — 2×2 quadrant on navy, hairline dividers, numbered. Reads as substance, not marketing.
  • How We Work — horizontal 5-node stepper with a connecting gradient line; collapses to a vertical timeline on mobile.
  • Outcomes — 3×2 list cards with a small directional icon each. Restrained — no invented metrics.
  • Credibility — 4 stat tiles over 3 reference columns + a logo row, all placeholder-friendly.
  • Final CTA — centered gradient card, single focused message, both CTAs repeated.

04 Mobile Layout Notes
  • Nav collapses to a hamburger at ≤920px; menu drops from the top as a full-width navy panel. CTA moves inside the menu.
  • Hero stacks to one column with the visual above the copy; orbital visual scales to ~78% width; floating chips reflow inside the frame.
  • Capabilities & Outcomes go 3 → 2 → 1 column as width drops (breakpoints ~900px and ~600px).
  • How We Work switches from a horizontal stepper to a vertical timeline with the connector on the left.
  • Why Us and Credibility columns stack to single column; stat tiles go 4 → 2.
  • Tap targets ≥ 44px; body text stays ≥ 15px; section padding compresses via clamp().
  • Modal is full-width with comfortable padding; form fields stack to one column under 480px.
05 Build Guidance

Ship it in your platform of choice.

Developer / hand-code fastest

The provided files are production-ready: one HTML, landing.css, landing.js, marks.js, and the assets/ folder. Drop on any static host (Netlify, Vercel, S3, GitHub Pages). Wire the form’s submit handler to your endpoint and you’re live.

Webflow

Recreate sections as native Webflow sections/grids. Add the three Google Fonts in Project Settings. Build the card/step grids with Grid or Flexbox. Use a Webflow Form for the modal (interaction: open on CTA click). Set the brand hexes as Swatches.

Framer

Use stacks for each section and a responsive grid for cards. Add fonts via the Text panel. The hero orbital can be a centered image + circular shapes. Use a Framer Form component and an overlay for the modal; enable scroll/appear effects to match the reveal animations.

WordPress

In a block builder (Gutenberg, Elementor, Bricks): build with columns/containers. Load fonts via the theme or a fonts plugin. Use a forms plugin (Gravity Forms, WPForms) in a popup for the modal. Or simply embed the static HTML build in a full-width page template.
